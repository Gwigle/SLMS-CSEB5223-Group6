# Student Learning Management System (SLMS)

## docs Folder Structure

This folder contains all **supporting documentation, reports, and diagrams** for the  
**Student Learning Management System (SLMS)** project.

It documents the **complete development lifecycle**, showing how the system evolved from initial design to final implementation.

---

## Contents

### Final Report
- `SLMS-CSEB5223-Group6-(Final Report).pdf`

Contains the **complete and finalized system**, including:
- Full system architecture
- Detailed module explanations
- Middleware API (Search Cache)
- Testing strategy and results
- Challenges, solutions, and improvements

This is the **main document for evaluation**

## Development Versions

### Version 2 – Initial Implementation
- `SLMS-CSEB5223-Group6-(Version2).pdf`

**Focus:**
- Basic system structure
- Initial class design (Course, Student)
- Core CRUD operations
- Simple menu-driven interface

**Limitations:**
- Limited validation
- No modular separation (managers not fully implemented)
- No relationship handling refinement
- No caching or advanced features

---

### Version 3 – System Enhancement
- `SLMS-CSEB5223-Group6-(Version3).pdf`

**Improvements from Version 2:**
- Introduced **Manager classes**
  - CourseManager
  - StudentManager
- Improved **code organization and modularity**
- Added **relationship handling (student–course mapping)**
- Enhanced **input validation**
- Improved menu structure and usability

**Remaining Gaps:**
- Middleware API not implemented yet
- Limited testing documentation
- No caching or suggestion features

---

### Version 4 – Feature Expansion
- `SLMS-CSEB5223-Group6-(Version4).pdf`

**Improvements from Version 3:**
- Introduced **SearchCacheManager (Middleware API)**
- Implemented:
  - Search caching
  - Auto-suggestions
- Strengthened **error handling**
- Improved **data consistency and validation**
- Refined **relationship management logic**
- Enhanced **user experience**

**Near Final Stage:**
- System structure stabilized
- Most features completed
- Ready for final testing and documentation

---

## Final Version – Complete System

**Enhancements over Version 4:**
- Comprehensive **testing strategy**
  - Unit testing
  - Integration testing
  - Scenario-based testing
- Improved **documentation quality**
- Added **CI practices and coding standards**
- Final refinement of:
  - Validation
